Micro Efficiencies

Oct 7, 2020 • 5 min read • Role: Product Designer • Company: Tealium
TL;DR: Transformed a mundane email invite process into a delightful experience by automatically parsing first and last names from 70% of corporate email addresses, reducing data entry friction and improving user onboarding efficiency.

One of my favorite things as a product designer is identifying ways to make a user's experience delightful. When an opportunity presents itself to add a little delight, particularly when the effort required is minimal, I get very excited. Such was the case when working on a feature for sending invites to users.

The Challenge #

In enterprise applications, administrators send invites to future users by entering email addresses. Traditionally, this requires the invited user to fill out their complete profile information upon registration—first name, last name, company, and title. But what if we could make this process more intelligent?

The insight came from one of my 4 Pillars of Enterprise Application Design: efficiency. Specifically, the principle of not requiring users to enter the same information twice and providing meaningful default values whenever possible.

Design Principle: In enterprise software, reducing friction in user onboarding directly impacts adoption rates and user satisfaction.

Data-Driven Discovery #

To validate my hypothesis that corporate email addresses often contain parseable name information, I analyzed our customer database. The results were compelling:

The analysis revealed that corporate email addresses commonly follow these patterns:

  • {first}.{last}@company.com
  • {first}-{last}@company.com
  • {first}_{last}@company.com
  • {first}.{middle}.{last}@company.com
  • {first}-{middle}-{last}@company.com
  • {first}_{middle}_{last}@company.com

Additionally, when users copy and paste email addresses from their address books, the format often includes the full name:

  • Joe Smith <smith@acme.com>
  • Jane Doe <jdoe@acme.com>

Implementation Approach #

The solution provides benefits at two critical touchpoints:

For Administrators #

Once an email is entered, a user record is created in the system. Without name parsing, administrators would only see email addresses in their user list, making it difficult to identify specific users. With automatic name extraction, the system displays meaningful names rather than cryptic email addresses.

For Invited Users #

When users click their invitation link, over 70% find their first and last names pre-populated in the registration form. They only need to add their job title, significantly reducing onboarding friction.

Design Considerations #

One critical design decision was whether to show the derived names to administrators during email entry. Displaying them as input fields might create a sense of obligation to fill or correct them, adding unnecessary work. Showing them as non-editable text could cause frustration.

The best interface is often the one that doesn't require any interaction at all.

The solution: Don't show the derived names during email entry. Let administrators focus on entering emails quickly. The names appear automatically in the system after submission, and administrators can edit them later if needed—but without any sense of obligation during the initial workflow.

Impact & Results #

This seemingly small enhancement had significant impact:

  • 70% reduction in form fields for the majority of users during registration
  • Improved administrator experience with meaningful user records instead of email-only entries
  • Zero additional effort required from administrators during the invite process
  • Increased delight when users see their names already filled in

Lessons Learned #

This project reinforced several key design principles:

  • Small, intelligent features can have outsized impact on user experience
  • Data analysis should drive design decisions
  • Sometimes the best UI is no UI—automation beats interaction
  • Consider the entire ecosystem when designing features (both admin and end-user perspectives)
